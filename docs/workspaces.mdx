# Workspace Management

SPT Teams provides powerful hierarchical workspace management that allows organizations to structure their operations across multiple levels, from main company workspaces down to regional offices and departments.

## üè¢ **Understanding Workspace Hierarchy**

### **Workspace Types**

SPT Teams supports a flexible hierarchical structure:

#### **Main Workspaces**
- **Purpose**: Top-level organizational units (companies, divisions)
- **Scope**: Full organizational control
- **Access**: Owners have complete administrative rights
- **Examples**: "SPT Corporation", "Marketing Division", "Operations Division"

#### **Sub-Workspaces**
- **Purpose**: Regional offices, branches, or specialized departments
- **Scope**: Limited to specific regions or functions
- **Access**: Assigned administrators with delegated authority
- **Examples**: "Accra Branch", "Kumasi Office", "Northern Region"

## üöÄ **Creating and Managing Workspaces**

### **Creating a Main Workspace**

:::info
**Owner Only**: Only users with owner privileges can create main workspaces.
:::

1. **Navigate to Workspace Management**:
   - Go to **Dashboard** ‚Üí **Workspaces**
   - Click **"Create New Workspace"**

2. **Configure Workspace Details**:
   - **Workspace Name**: Enter a clear, professional name
   - **Description**: Provide a brief description of the workspace purpose
   - **Type**: Select organization type (Company, Nonprofit, Government, Education, Other)
   - **Logo**: Upload workspace logo (optional)

3. **Set Workspace Settings**:
   - **Default Currency**: Choose primary currency (GHS, USD, EUR, etc.)
   - **Time Zone**: Set workspace time zone
   - **Date Format**: Choose date display format
   - **Language**: Select primary language

4. **Configure Advanced Settings**:
   - **Allow Admin Workspace Creation**: Enable/disable admin sub-workspace creation
   - **Cross-Workspace Access**: Configure data sharing between workspaces
   - **Security Settings**: Set authentication and access requirements

### **Creating Sub-Workspaces**

1. **Access Sub-Workspace Creation**:
   - Go to **Workspaces** ‚Üí **Manage Workspaces**
   - Select the parent workspace
   - Click **"Create Sub-Workspace"**

2. **Configure Sub-Workspace**:
   - **Name**: Enter sub-workspace name (e.g., "Accra Branch")
   - **Parent Workspace**: Confirm parent workspace selection
   - **Region**: Assign to specific region
   - **Branch**: Assign to specific branch
   - **Description**: Explain sub-workspace purpose

3. **Assign Administrator**:
   - **Admin User**: Select user to manage the sub-workspace
   - **Permissions**: Set admin permissions and scope
   - **Access Level**: Define what the admin can manage

4. **Configure Inheritance**:
   - **Settings Inheritance**: Choose which settings to inherit from parent
   - **Permission Inheritance**: Configure permission inheritance rules
   - **Data Access**: Set data sharing between parent and sub-workspace

## üîÑ **Workspace Switching and Navigation**

### **Switching Between Workspaces**

#### **For Owners**
1. **Workspace Selector**:
   - Click the workspace name in the top navigation
   - Select from dropdown list of accessible workspaces
   - View workspace type and hierarchy level

2. **Quick Navigation**:
   - Use keyboard shortcuts for workspace switching
   - Access recent workspaces from quick menu
   - Pin frequently used workspaces

#### **For Admins and Members**
1. **Assigned Workspaces**:
   - View only workspaces you have access to
   - Switch between assigned workspaces
   - See workspace hierarchy and relationships

### **Cross-Workspace Management**

#### **Owner Cross-Workspace Features**
- **Unified Dashboard**: View aggregated data across all workspaces
- **Cross-Workspace Analytics**: Generate reports spanning multiple workspaces
- **Bulk Operations**: Manage multiple workspaces simultaneously
- **Data Migration**: Move data between workspaces

#### **Admin Sub-Workspace Management**
- **Local Administration**: Manage assigned sub-workspace independently
- **Data Isolation**: Keep sub-workspace data separate from parent
- **Reporting Up**: Submit reports and data to parent workspace
- **Permission Delegation**: Assign permissions within sub-workspace scope

## üìä **Workspace Data Management**

### **Data Scoping and Isolation**

#### **Main Workspace Data**
- **Global Settings**: Organization-wide configurations
- **User Management**: Complete user directory
- **System Administration**: Full administrative controls
- **Cross-Workspace Analytics**: Aggregated data and insights

#### **Sub-Workspace Data**
- **Local Settings**: Sub-workspace specific configurations
- **Local Users**: Users assigned to the sub-workspace
- **Local Projects**: Projects created within the sub-workspace
- **Local Reports**: Reports generated within the sub-workspace

### **Data Inheritance and Sharing**

#### **Inheritance Rules**
- **Settings Inheritance**: Sub-workspaces inherit parent settings by default
- **Permission Inheritance**: Users inherit permissions from parent workspace
- **Template Inheritance**: Report templates and forms can be inherited
- **Configuration Inheritance**: System configurations flow down the hierarchy

#### **Data Sharing Options**
- **Full Access**: Complete data visibility across hierarchy
- **Selective Sharing**: Choose specific data to share
- **Read-Only Access**: View-only access to shared data
- **No Sharing**: Complete data isolation

### **Workspace-Level Permissions**

#### **Owner Permissions**
- **Full Access**: Complete control over all workspaces

- **User Management**: Can assign users to any workspace

- **Sub-Workspace Management**: Manage assigned sub-workspaces

#### **Member Permissions**
- **Workspace Access**: Access to assigned workspaces only
- **Data Access**: Access to data within assigned scope
- **Feature Access**: Access to features based on role and permissions

### **Cross-Workspace Security**

#### **Data Isolation**
- **Workspace Boundaries**: Clear separation between workspace data
- **Access Controls**: Strict permission-based access
- **Audit Trails**: Complete logging of cross-workspace activities
- **Data Encryption**: All data encrypted at rest and in transit

#### **Permission Inheritance**
- **Hierarchical Permissions**: Permissions flow down the hierarchy
- **Override Capabilities**: Local permissions can override inherited ones
- **Granular Control**: Fine-grained permission management
- **Temporary Access**: Time-limited cross-workspace access
